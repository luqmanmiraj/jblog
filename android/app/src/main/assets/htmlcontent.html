Summary: ​Writing stories in Storybook involves defining variations and states of UI components in isolation for development, documentation, and testing purposes.

​When writing stories in Storybook, you define different states or variations of your UI components. Each story represents a specific configuration or usage scenario of your component.


Step-by-step guide on how to write stories in Storybook:

Step 1: Create a Storybook story file

Create a new file (e.g., ComponentName.stories.js) alongside your component file in the same directory. This file will contain the stories for your component.

Step 2: Import necessary dependencies

At the top of your story file, import the required dependencies, including React and your component.

import React from 'react'; 
import YourComponent from './YourComponent';


​Step 3: Define the stories using the storiesOf function
Use the storiesOf function to define your stories. This function takes the component's name as the first argument and returns an object on which you can chain different story methods.


import { storiesOf } from '@storybook/react'; 
 
storiesOf('YourComponent', module) 
  // Define your stories here 
  .add('Story 1', () => <YourComponent prop1="value1" />) 
  .add('Story 2', () => <YourComponent prop2="value2" />);


​In the above example, we use storiesOf to define stories for the YourComponent component. The add method is then used to create individual stories, each with a unique name and a render function that returns JSX representing the component with specific props.

Step 4: Export the story file
At the end of the story file, export it as the default export.


​export default { 
  title: 'YourComponent', 
};

​The title property in the exported object sets the title for the group of stories. In this case, it will be displayed as "YourComponent" in the Storybook sidebar.

Step 5: View your stories in Storybook

Run your Storybook server using the appropriate command (npm run storybook or yarn storybook). Open the Storybook interface in your browser, and you should see your component's stories listed under the specified title.

You can add as many stories as needed, covering different use cases and variations of your component. Each story can have its own set of props and render logic.


Official documentation: https://storybook.js.org/docs/react/writing-stories/introduction
